# Video Capture Checklist

Use this while recording the ETHGlobal demo video.

## Target Length

Aim for 2 to 4 minutes.

## Browser Tabs

Open these before recording:

1. Live app: https://dreamnet-quorum-lab-ethnyc.dreamnet-intel.workers.dev
2. GitHub repo: https://github.com/BrandonDucar/dreamnet-quorum-lab-ethnyc
3. Walrus read URL: https://aggregator.walrus-testnet.walrus.space/v1/blobs/7aOiEmqACMCHzOlw6BN7fKWIvDi37knw6F5OtWpJ6ec

## Opening Line

```text
This is DreamNet Quorum Lab, a clean-room ETHGlobal New York 2026 build for auditable AI agent quorum receipts.
```

## Show In Order

1. The live page hero and product framing.
2. The form with market, instrument, horizon, and scenario.
3. A generated 31-agent receipt.
4. The strict 28-of-31 threshold and the no-execution safety boundary.
5. ENS live resolver with `vitalik.eth`.
6. World gate panel showing proof-required status.
7. Walrus storage panel and verified blob link.
8. GitHub repo with public commit history.

## Say Clearly

```text
This is not a trading bot. It has no wallet signing path, no broker path, and no autonomous execution path.
```

```text
ENS names the agents. World verifies the human boundary. Walrus stores the receipt as retrievable evidence.
```

```text
The core idea is that agents should not just answer. They should produce a receipt that shows who voted, whether enough agents agreed, and what the system is not allowed to do.
```

## Do Not Say

- Do not say the DreamNet ENS subnames are already registered unless they are activated during the event.
- Do not say World proof is live until credentials are configured and a proof succeeds.
- Do not say the system trades, spends, signs, posts, or executes.
- Do not say sponsors approved the project unless a sponsor says so directly.

## Final Line

```text
DreamNet Quorum Lab is the smallest public wedge for the larger Agent OS: identity, governance, receipts, and human-controlled autonomy for persistent AI agents.
```
